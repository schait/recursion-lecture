When you call a function, you “freeze” where you are until that function returns, and then continue where you left off.

So a prints hello, calls b which prints world, calls c which prints i and returns back to b which then prints love which then returns back to a which prints python.

Recursion Analogy: Nesting Dolls¶

  • Each doll represents a function call

  • As function calls itself, it pops open to reveal a new doll

  • As recursive functions complete, dolls pop back together (in reverse order)

The Call Stack¶

Count to 3¶

It turns out, any loop can be written instead with recursion (and vice versa).

Using a while loop:

def count_loop():
    """Count to 3, using a while loop."""

    n = 1

    while n <= 3:
        print(n)
        n = n + 1

Using recursion:

def count_recursive(n=1):
    """Count to 3, using recursion."""

    if n > 3:
        return

    print(n)
    count_recursive(n + 1)

Recursion Requirements¶

Base Case¶

def count_recursive(n=1):
    """Count to 3, using recursion."""

    if n > 3:
        return

    print(n)
    count_recursive(n + 1)

  • Every recursive function needs a base case

    • How do we know when we’re done?

    • Like a stop sign

Explicit vs. Hidden Base Cases¶

def count_recursive(n=1):
    """Count to 3, using recursion."""

    if n > 3:
        return

    print(n)
    count_recursive(n + 1)

def count_recursive_alt(n=1):
    """Print numbers from 1 to 3."""

    if n <= 3:
        print(n)
        count_recursive_alt(n + 1)

Progress¶

We have to be moving forward toward the base case.

def count_recursive(n=1):
    """Count to 3, using recursion."""

    if n > 3:
        return

    print(n)
    count_recursive(n + 1)

What If There’s No Base Case?¶

def count(n=1):
  """Print numbers from 1 to forever."""

  print(n)
  count(n + 1)

>>> count()

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
  File "<stdin>", line 3, in count

The same error will happen if you have a base case but don’t make progress toward it.

This error is also called a Stack Overflow error.

Strategies for Recursion¶

Example¶

Using recursion, find the sum of a list of numbers. (You can’t use the built-in sum() function.)

What’s the Base Case?¶

Often a base case is an “empty case” — e.g. an empty array, None, a leaf in a tree, etc.

Here, a good base case would be an empty list.

Make the Problem Smaller and Smaller¶

Many recursion problems involve “breaking off a piece” and making the problem smaller and smaller.

  • sum of numbers = first number + sum of rest of numbers

  • sum([1, 2, 3]) →

  • 1 + sum([2, 3]) →

  • 1 + 2 + sum([3]) →

  • 1 + 2 + 3 + sum([]) ← base case: empty list!

Strategy #1: Top-Down / “Backpack”¶

def sum_nums_1(nums, total=0):
    """ Sum list of numbers recursively using "backpack" or "top down" strategy """

    if not nums:
        return total

    remaining_nums = nums[1:]
    running_total = total + nums[0]

    return sum_nums_1(remaining_nums, running_total)

  • Add a parameter (with a default value) to store the result at each step — the “backpack” variable

  • When you break off a piece and make a recursive call, update this variable

  • When you reach the base case, return this variable

See this code in Python Visualizer

Strategy #2: Bottom-Up / “Breadcrumbs”¶

def sum_nums_2(nums):
    """ Sum list of numbers recursively using "breadcrumbs" or "bottom up" strategy """

    if not nums:
        return 0

    first_num = nums[0]
    sum_of_remaining = sum_nums_2(nums[1:])

    return first_num + sum_of_remaining

  • When you make a recursive call, leave a “breadcrumb” to pick up on the way back

  • After returning from each function call, pick up the “breadcrumbs”

  • In this example, return 0 when you reach the base case, because the sum of an empty list is 0

List Doubler¶

The Problem¶

“For every item in a list, print the value, doubled”

lst = [ 1, 2, 3 ]  #  => 2 4 6

def doubler_basic(lst):
    """Print every item in list, doubled."""

    for item in lst:
        print(item * 2)

The Challenge¶

  • Some items can be lists themselves

  • We want to “flatten” them and still print doubled

    lst = [ 1, [2, 3], 4 ]  #   => 2 4 6 8

def doubler_flattener(lst):
    """Print items in flattened list, doubled, for 1 level of nesting"""

    for original_list_item in lst:
        if isinstance(original_list_item, list):
            for sublist_item in original_list_item:
                print(sublist_item * 2)
        else:
            print(original_list_item * 2)

Oh No!¶

Some of those items can be lists!

lst = [1, [2, [3], 4], 5]  # => 2 4 6 8 10

def doubler_flattener_more(lst):
    """Print items in flattened list, doubled, for 2 levels of nesting."""

    for original_list_item in lst:
        if isinstance(original_list_item, list):
            for sublist_item in original_list_item:
                if isinstance(sublist_item, list):
                    for nested_item in sublist_item:
                        print(nested_item * 2)
                else:
                    print(sublist_item * 2)
        else:
            print(original_list_item * 2)

Arbitrary Depth with Loop¶

def doubler_no_recursion(lst):
    """Print items in list doubled, not using recursion."""

    stack = list(reversed(lst))

    while stack:
        current_item = stack.pop()
        if isinstance(current_item, list):
            # If it's a list, add it to stack, reversed
            for inner in reversed(current_item):
                stack.append(inner)
        else:
            print(current_item * 2)

It works, but it’s pretty difficult to understand!

This solution uses a stack data structure to add new items to the end and remove them from the same side so we always remove the item most recently added.

This code is worth studying and reasoning about even though this problem is more better suited for recursion.

Recursively¶

def doubler(lst):
    """Print items in list doubled, using recursion."""

    for item in lst:
        if isinstance(item, list):
            doubler(item)
        else:
            print(item * 2)

Trees¶

A data structure where one item points to many items

class Node:
    """Tree node."""

    def __init__(self, data, children=[]):
        self.data = data
        self.children = children

    def __repr__(self):
        return f"Node {self.data}>"

resume = Node("resume.txt")
recipes = Node("recipes.txt")
jane = Node("jane/", [resume, recipes])
server = Node("server.py")
jessica = Node("jessica/", [server])
users = Node("Users/", [jane, jessica])
root = Node("/", [users])

def list_nodes(node):
    """List all nodes."""

    print(node.data)
    for child in node.children:
        list_nodes(child)

def indent_list(node, depth=0):
    """List all nodes, indented."""

    print("  " * depth + node.data)
    for child in node.children:
        indent_list(child, depth + 1)
